1.命令的基本格式：

命令提示符：

[root@localhost ~] #

root  当前登陆用户
localhost  主机名
~  当前所在目录(家目录)，家目录：用户初始目录，管理员的家目录是根目录下的root目录(/root)，普通用户的家目录是home目录下的当前用户目录(/home/user1)。
#  超级用户的提示符，普通用户的提示符是$

命令格式：

命令 [选项] [参数]

注意：个别命令使用不遵循此格式，当有多个选项时，可以写在一起(如：-a -l 等于 -al)，简化选项与完整选项(如：-a 等于 --all)。

2.ls [选项] [文件或目录]

选项：
-a 显示所有文件，包括隐藏文件
-l 显示详细信息
-d 查看目录属性(看目录本身的信息，而不是下属文件)
-h 人性化显示文件大小
-i 显示inode(显示i节点，查看每个文件的id号)

如果没有设置文件则默认为当前所在目录。

ls -l，显示如下：

-rw-------. 1 root root  1289 3月  26 06:04 anaconda-ks.cfg
-rw-r--r--. 1 root root 27199 3月  26 06:04 install.log
-rw-r--r--. 1 root root  9043 3月  26 06:02 install.log.syslog

第一列：权限

默认是10位，- 文件类型(- 文件 d 目录 l 软链接文件)，
2-4位，代表所有者，u
5-7位，代表所属组，g
8-10位，代表其他人，o
r代表读，w代表写，x代表执行

后面的点代表ACL权限。

第二列：数字，引用计数。

第三列：所有者。

第四列：所属组。

第五列：文件大小，单位是字节。如果加h表示用人看得懂的方式显示(统计都可以加h)。

第六列：最后一次修改时间。

第七列：文件名。

3.目录处理命令：

目录文件：用于保存文件，相当于windows中的文件夹。

建立目录：mkdir
mkdir -p [目录名]
-p：递归创建，指定父目录不存在时一并创建。

切换所在目录：cd
cd [目录]
简化操作：
cd ~ 进入当前用户的家目录
cd 
cd - 进入上次目录
cd .. 进入上一级目录
cd . 进入当前目录

ctrl + l ：清屏
tab：目录补全和命令补全。(有多个可能时按两次，显示所有符合的目录或命令)

相对路径：参照当前所在目录进行查找。
绝对路径：从根目录开始指定，一级一级递归查找。

查询所在目录位置：pwd

删除空目录：rmdir，只能删除空目录(很少用)。
rmdir [目录名]

删除文件或目录：rm
rm -rf [文件或目录] 不加选项时会提示是否确认删除
-r 删除目录
-f 强制

复制：cp
cp [选项] [原文件或目录] [目标目录]
选项：
-r 复制目录
-p 连带文件属性复制
-d 若源文件是链接文件，则复制链接属性
-a 相当于 -pdr (目标文件和原文件所有属性都一样，一般用这个)
复制目标目录后跟文件名，表示复制之后取给定的新文件名。
注意：ll等于ls -l

剪切或改名命令：mv

mv [原文件或目录] [目标目录]
如果源文件目录和目标目录相同，那么此命令就是改名命令。

4.常用目录的作用：

/  根目录
/bin  命令保存目录(普通用户就可以读取的命令)  根和usr下的bin和sbin，这四个目录都是用来保存系统命令的。
/boot  启动目录，启动相关文件
/dev  设备文件保存目录
/etc  配置文件保存目录
/home  普通用户的家目录
/lib  系统库保存目录(函数库)
/mnt  系统挂载目录
/media  挂载目录
/misc  挂载目录
后三个是空目录，用来挂载外设。可以在mnt中建不同的目录来挂载usb/光盘等等。
/root  超级用户的家目录
/tmp  临时目录，临时数据可以放在这里
/sbin  命令保存目录(超级管理员才能使用的目录)
/proc  直接写入内存
/sys  
上面两个目录不能直接操作，保存的是内存的过载点。
/usr  系统软件资源目录
	/usr/bin  系统命令(普通用户)
	/usr/sbin  系统命令(超级用户)
/var  系统相关文档内容

推荐随便使用的目录是家目录和临时目录，其他目录在有需要的时候再使用。

5.链接命令：ln

ln -s [原文件] [目标文件]
-s 软链接

硬链接特征：(可以理解为一间教室两个门)，添加硬链接会增加引用数，创建硬链接不好
拥有相同的i节点和存储block块，可以看作是同一个文件。
可通过i节点识别
不能跨分区
不能针对目录使用
只要还有其中一个文件就可以使用


软连接特征：类似于Windows的快捷方式。推荐
软连接拥有自己的i节点和block块，但数据块中只保存文件的文件名和i节点号，并没有实际的文件数据。
Irwxrwxrwx I 软连接 ，软链接文件的权限都是rwxrwxrwx
修改任意文件，另一个都改变
删除原文件，软链接不能使用
做软链接一定要写具体的源文件路径。

6.文件处理命令：

创建空文件命令：touch
touch 文件名
如果文件名中包含空格应该用""包裹起来

显示文件内容命令：cat，仅适合内容较少的文件
cat [-n] 文件名
-n 表示显示文件行号

反向显示文件内容命令：tac，内容从下往上显示，仅适合内容较少的文件
tac 文件名

分页显示文件内容：more，不能向前翻页
more 文件名
空格或f翻页，Enter换行，q或Q退出。

分页显示文件内容命令：less，可前后翻页(用得最多)
less 文件名
空格/f/PgDn，向后翻页
PgUp，向前翻页
Enter或向下箭头，一行一行往后显示
向上的箭头，一行一行往前显示
q或Q退出
/搜索字符然后回车，搜索。

显示文件内容命令：head，指定行数
head [-n 数量] 文件名
显示n，默认为20

反向文件内容命令：tail，文件即时更新后也能动态显示。
tail -n 20 -f 文件名
-n 指定行数
-f 动态显示
ctrl + c 退出

7.文件搜索命令：

locate：文件搜索命令
locate 文件名
在后台数据库中按文件名搜索，搜索速度快。
搜索的是/var/lib/mlocate下的后台数据库，这个数据库是每天更新的。使用updatedb立即更新数据库，就可以查到新建的文件了。
但是在/etc/updatedb.conf配置文件中定义了一些规则，可能导致某些目录不搜索。


命令搜索命令：

whereis 命令名
搜索命令所在路径及帮助文档所在位置(只能查系统命令)
选项：
	-b  只查找可执行文件
	-m  只查找帮助文件
	
which 命令名
和whereis很类似，可以看到命令所在位置，如果有别名还可以看到别名。

$PATH：环境变量，定义的一组系统搜索命令的路径。和windows一样。


文件搜索命令find：强大，但耗费资源。经常使用。
find [搜索范围] [搜索条件]
应该避免大范围的搜索。

搜文件(完全匹配)：
find /tmp/test/test1 -name 123.txt

通配符：
*匹配任意内容，?匹配任意字符，[]匹配任意中括号内的字符。

搜文件(模糊匹配)：使用通配符
find /tmp/test/test1 -name "123.*"
find /tmp/test/test1 -name "123.tx?"
find /tmp/test/test1 -name "123.tx[tk]"

选项使用
-iname  不区分大小写
-user  按所有者搜索
-nouser  查找所有没有所有者的文件  proc和sys下的文件可能由内核生成，没有所有者，外设在windows中创建的文件也可能没有所有者，其他无所有者的文件是垃圾文件，应该被清除。

选项使用
-10 10 +10 10天内 10天当天 10天前
atime ctime mtime 文件访问 改变文件属性 修改文件内容
find /var/log/ -mtime +10 10天前修改的文件

选项使用
-size 按文件大小查找 -20 20 +20 小于 等于 大于
find /root -size 25k
find /root -size 20M

按照i节点来搜索
find /root -inum 262421

复杂查找：
find /root -size +20k -a -size -50k  root下大于20k小于50k的文件(-a且的意思，-o或的意思)
find /root -size +20k -a -size -50k -exec ls -lh（对前面的命令结果执行第二条命令） {} \; 对上面的结果详细显示，这里放的第二命令只能是可以执行的。


搜索字符串命令grep：
grep [选项] 字符串 文件名
在文件当中匹配符合条件的字符串,grep可以使用正则表达式来进行匹配
选项：
	-i  忽略大小写
	-v  排除指定字符串
grep "si" install.log
